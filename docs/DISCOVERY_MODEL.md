# Discovery model

Discovery is a secondary path under People, not a separate product universe.

## Ranking inputs

- Existing relationship and repeat co-listening history.
- Current public availability.
- Shared artists and recent listening patterns.
- Mutual social context when available.
- Broad distance bands only after explicit location consent.

## Presentation

Show evidence such as shared artists, similar listening time, or mutual Tethers. A confidence label may communicate how much information supports the explanation. Do not present invented percentages or add random noise to a displayed score.

## Privacy

Ghost users are excluded. Block relationships are enforced in both directions. Phone numbers, exact coordinates, private tokens, dating answers, and raw model vectors do not appear in discovery responses.
