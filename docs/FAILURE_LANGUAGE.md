# Failure language

Failures should tell the user what happened, whether the other person is still available, and the next safe action.

Examples:

- “We found a different recording on Apple Music. Review it before joining.”
- “Maya is still listening, but synchronization timed out. Try again.”
- “This session requires a Knock. Maya will choose whether to open it.”
- “Your provider disconnected. Reconnect without losing the invitation.”

Avoid generic “Something went wrong” copy when a bounded failure code is known.
