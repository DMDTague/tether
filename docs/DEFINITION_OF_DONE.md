# Definition of done for a core Tether change

A core change is not done until:

- The user can understand the action without prior vocabulary training.
- The ten-second shared-listening path is not made slower.
- Open Door, Knock First, and Ghost behavior remains correct.
- Failure and recovery states are explicit.
- Required product events are versioned and privacy-safe.
- Accessibility and reduced-motion behavior are verified.
- Automated tests cover the trust or reliability boundary affected.
- Documentation describes what is actually committed.
