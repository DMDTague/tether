# Memory model

Tether preserves relationships rather than posts.

A Memory Anchor may contain:

- The people who listened.
- The canonical track or verified provider versions.
- Session time and duration.
- Relational actions such as Pulse and queue contribution.
- An optional mood or short note.
- A deterministic Tether artwork fallback.

Anchors should be created automatically after meaningful sessions. Users may edit or delete their personal annotation and should be able to export or delete their memory history.

Time Capsules remain intentionally asynchronous: a person leaves a song and context for a later moment rather than competing for feed reach.
