# Accessibility baseline

- Primary controls require accessible names and sufficient target size.
- Metadata should ordinarily remain at or above 13px.
- Color cannot be the only indicator of Open Door, Knock First, Ghost, sync, or error state.
- Native dialog semantics or equivalent focus trapping are required.
- Reduced-motion preferences disable ambient and decorative movement.
- Pulse haptics must have a visible equivalent.
- Track, artist, participant, and synchronization state remain readable without artwork.
- Keyboard and switch-control navigation must reach all core actions in a predictable order.
