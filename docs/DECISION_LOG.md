# Decision log

- Treat Tether as a shared-listening utility rather than a general music social network.
- Use Listen, People, and You as the primary architecture.
- Make the shared Stage the flagship experience.
- Fold messages into People and remove Exchange from primary navigation.
- Rank relationships and explain evidence instead of displaying fabricated percentages.
- Enforce Open Door, Knock First, and Ghost as backend trust states.
- Quantize location before storage and return broad bands only.
- Keep core listening free of ad gates.
- Measure Meaningful Shared Listens rather than generic engagement.
- Describe the repository honestly as a backend plus browser prototype until a mobile client is committed.
